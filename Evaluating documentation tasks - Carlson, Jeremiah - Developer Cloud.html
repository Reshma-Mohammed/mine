Evaluating documentation tasks

Skip to end of metadata
  • Created by Mahajan, Natasha, last modified by Hegde, Manisha Dayananda on May 27, 2019
Go to start of metadata

[ 1Use Cases ] [ 2Topic Types ]

This topic demonstrates how to determine the impact on documentation for a User Story or a Defect.

Use Cases

Use Case

Topics/sections likely to be created/modified

A family has been created.

  • Concept
  • Task
  • Family Field Description
  • Data Model diagram
  • System Code Tables (if they have been created to populate the fields in the family)
  • Security Groups
  • Data Loader Workbook Layout and Use (if the Data Loader can be used to import the family records)

A family field has been created or deprecated, or the field behavior has been modified.

  • Family Field Description
  • System Code Tables (if one is used to populate the field)
  • Data Loader Workbook Layout and Use (if the field is added to the Data Loader workbook)

A page or section has been added.

  • Concept
  • Task
  • Screenshots
  • URLs
  • Catalog Items (if they have been created)

A UI label has been modified.

  • Screenshots
  • References to the UI label
  • Family Field Description

A catalog item has been added, modified, or deleted.

  • Catalog Items

Topic Types

Topic Type

Description

Examples

Release Note

Provides information about what has changed in the current release compared to the previous release and how it affects a user.

Note: For more information, refer to Release Notes.

  • New Features
  • Enhancements
  • Resolved Issues
  • Obsolete Features
  • Known Issues
  • Limitations

Concept

Provides information about what something is, what it does, why someone could use it, and how a workflow meets a certain use case.

  • About Test Equipment
  • About Creating a Route

Note: This concept topic also contains a workflow.

  • Overview of the Calibration Management Module

Task

Provides step-by-step instructions to perform a task.

This topic includes the following information, as applicable or necessary:

  • Prerequisites.
  • Purpose of the task, if it is not self-explanatory.
  • Sequential steps.
  • Step results.
  • Screenshots.
  • Information about an alternative method of performing the task.
  • Task result.
  • A brief description of what the user must do immediately after performing the task (that is, next steps).
  • Create an Operation
  • Generate a Report

Workflow

Provides the high-level steps for using an application or for performing a procedure that contains multiple tasks.

  • HAZOP Analysis Workflow

Note: This is the workflow topic for using a module.

  • Define Component

Data Model

Provides a diagram that represents the relationships between the families that are used in an application.

  • eLog Data Model

Data Loader

Provides information about the data loader. This topic is a collection of the following subtopics:

  • Data Loader Introduction
  • Data Loader Requirements
  • Data Loader Data Model
  • Data Loader General Loading Strategy
  • Data Loader Workbook Layout and Use
  • Data Loader Load Verification
  • Thickness Monitoring Data Loaders
  • GAA Wind Data Loaders

Calculation

Provides the method to calculate a value used in an application.

Note:

  • You can include an example in this topic.
  • If the calculation information is intellectual property, you must inform the technical writer. (The technical writer will ensure that the confidential information is accessible only to the intended audience.)
  • Calculating Inventory Group Mass
  • Calculating Inventory Component Mass
  • Calculate Unaccounted Loss

First-Time Deployment

Provides step-by-step instructions to deploy an application for the first time.

  • Deploy RB1 581 for the First Time
  • Deploy GAA Wind for the First Time

Upgrade

Provides step-by-step instructions to upgrade an application to the current version.

Note:

  • For each previous version, create a separate section in the topic.
  • You can include any information that the customer must know before upgrading the application.
  • Calibration Management Upgrade

Security Groups

Provides a table of baseline Security Groups and Security Roles for an application, and a table of privileges that exist for each Security Group for each family used in the application.

  • Calibration Management Security Groups

Catalog Item

Provides information about the Catalog item (such as Dashboards, Graphs, and Queries) used in an application.

Note: Each folder in the Catalog will have its own topic.

  • eLog Catalog Items
  • GAA Wind Catalog Items

System Requirements

Provides hardware and software requirements (such as license and components) for the application to function.

  • GAA Wind System Requirements

Family Field Description

Provides information about the fields that exist for a family in an application.

Note:

  • You can exclude internal key fields from this topic.
  • This topic helps a user specify values in the fields in the application.
  • GAA Wind Family Field Descriptions

Example

Provides an example for a use case in an application.

Note: You can use this topic to explain a complex functionality.

  • PLA Examples

Flow Diagrams

Provides a diagram that represents the sequence of operations in an application or the relationships between the various components in the application (for example, information flow between the services that run in the background, system architecture diagram).

  • About Information Flow

URLs

Provides information about the URLs used in an application, along with an example.

Note: This topic helps a user create or modify a query or dashboard.


  • GAA Wind URLs

System Code Tables

Provides information about the System Code Tables used in an application.

Note: This topic helps a user add or remove items from a drop-down list box.


  • GAA Wind System Code Tables

Site Filtering

Provides the site filtering information for the families in an application, along with an example.

Note: This topic helps users identify the types of records that they can access.

  • Calibration Management Site Filtering
  • GAA Wind Site Filtering

State Management

Provides information about each baseline state in the state management workflow, along with a diagram.

  • PLA State Management

Scripts

Provides the pre-defined scripts to perform a task.

  • Table Deletion Script for Workflow Version 2.2

Mapping

Provides information about the mapping between two or more entities in the application (such as family fields and device fields).


  • About Values Mapped to RBI Recommendations

Other Reference Information

Provide any other fact-based information that could support users as they perform a task or that they could refer to when needed.

  • Pages and Windows
  • Relationship
  • Reports
